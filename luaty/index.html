Luaty

Luaty is an indent sensitive language with partial type inference that compiles to Lua.

Overview

Luaty stands for Lua with more or less typing. Typing has two meanings here, both are the features of Luaty:

Type inference

Luaty type checker statically infers variable types and warns when they are used inconsistently in the code. Although it is impossible to correctly and fully infer all variable types, some warnings are quite helpful in uncovering common coding mistakes or typos.

Type less

Due to offside syntax, Luaty could use less syntax boilerplate than Lua. end, do, then are removed without causing ambiguity. Luaty codes appear shorter and cleaner, yet maintains straightforward mapping to its Lua counterpart.

That said, Luaty strives to limit its syntactic changes. Lua language is already very well designed, so the rule of thumb is that, if something can be achieved by introducing a function, it should not be a syntactic feature of Luaty. That means Lua users can be productive with Luaty in an afternoon, and Lua code can be hand converted into Luaty easily using a Find/Replace editor.

In short, it boils down to: we type check, you type less

Usage

Luaty only requires LuaJIT to run.

With LuaJIT in your path, clone the Luaty repo, and cd into it.

Indent rules

  1. Either tabs or spaces can be used as indent, but not both in a single file.
  2. Comments have no indent rule.
  3. Blocks such as if, for, while, do and lambda expression -> can have child statement(s).
    • A single child statement may choose to stay at the same line as its parent
    • Multiple child statements must start at an indented newline
  4. A table constructor or function call can be indented, but the line having its closing brace/parenthesis must realign back to its starting indent level.
  5. A semicolon ; is used to terminate single-lined function if it causes ambiguity when used in a list of expressions.
